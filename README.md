### 5/20
- [INDEX]に\<ol\>要素を配置したが\<h2\>要素より右方向へずれていた。開発者ツールで確認すると\<ol\>要素は独自にpaddingが充てられておりそれを調整する必要があった。
- メインイメージはmax-width:1000pxだがコンテンツは600pxだった。コンテンツの背景色を1000pxに維持して幅は600にするのに苦労した。

### 5/21
- height: auto;を指定すると画像が縦横比を保ったまま縮小する
- 要件：メインは1000px、コンテンツは600px。max-width:1000とwidth:100%を組み合わせると画面幅1000未満になっても画面内に収まる
- \<img\>要素はインライン要素なので「margin: 0 auto;」を使って位置をセンターにするにはdisplay:blockを組み合わせないといけない
- メインコンテンツ全体をラッピングするのはNGでレイアウトが制御しづらくなる。セクションごとに必要なラッピングを充てること。
- \<ol\>要素の\<li\>はテキストをセンターにすると番号が左に取り残される。\<ol\>のデフォルトのpadding-leftが大きいのが原因。これを調整していくが小さくしすぎると小さい画面でテキストの文字が番号の下に回り込む可能性がある。
list-style-position: inside or outside;も一緒に見ていく。
